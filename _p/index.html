📋 管理方式变更说明

自 v5.0 起，博客内容改为 JSON 文件驱动，不再使用网页端登录发布。

所有博客数据存储在 data/posts.json 文件中，通过 Git 推送到 GitHub 即可更新网站内容。

🔄 更新流程

1 编辑 data/posts.json 文件，添加/修改博客文章

2 本地预览确认无误

3 提交并推送到 GitHub：

git add data/posts.json
git commit -m "update: 更新博客内容"
git push origin main

4 等待 GitHub Pages 自动部署（约1-2分钟）

📝 JSON 数据格式

{
  "version": "4.0",
  "updatedAt": "2026-07-08T00:00:00Z",
  "posts": [
    {
      "id": 1,
      "title": "文章标题",
      "date": "2026-07-08",
      "content": "文章正文内容...",
      "tags": ["标签1", "标签2"],
      "image": "图片URL（可选）"
    }
  ]
}

💡 提示：id 必须唯一，date 格式为 YYYY-MM-DD，tags 为字符串数组。
